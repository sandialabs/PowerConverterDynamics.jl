API

PowerConverterDynamics.ConverterParams — Type
ConverterParams{T<:Real}

Parameter set for the averaged DC/DC converter + DC bus + battery dynamic model.

The sign convention is:

  • Positive source current moves power from the source side to the DC bus.
  • Positive battery current discharges the battery into the DC bus.
  • Positive load current draws current from the DC bus.
PowerConverterDynamics.ConverterParams — Method
ConverterParams(; kwargs...)

Create ConverterParams with numerically stable defaults.

Defaults target a high-power DC bus and are intended as a starting point, not a tuned design.

PowerConverterDynamics.FlowSnapshot — Type
FlowSnapshot{T<:Real}

Algebraic values evaluated from a state/input pair.

PowerConverterDynamics.SimulationResult — Type
SimulationResult

Outputs from simulate:

  • time: simulation timestamps.
  • states: dynamic state at each timestamp.
  • flows: algebraic quantities at each timestamp.
PowerConverterDynamics.StateDerivative — Type
StateDerivative{T<:Real}

Time derivatives for SystemState.

PowerConverterDynamics.SystemInput — Type
SystemInput{T<:Real}

External inputs at one simulation instant:

  • source_voltage: source-side DC voltage (V).
  • source_current_command: commanded source current into converter (A).
  • source_current_available: available source current limit from upstream module (A).
  • load_current: bus load current demand (A).
  • battery_current_bias: dispatch bias for battery current (A); positive discharges to bus.
PowerConverterDynamics.SystemInput — Method
SystemInput(; kwargs...)

Create a typed input object.

PowerConverterDynamics.SystemState — Type
SystemState{T<:Real}

Dynamic states:

  • bus_voltage: DC bus voltage in volts.
  • soc: battery state-of-charge fraction in [0, 1].
  • controller_integrator: integral state used by the battery bus-voltage controller.
PowerConverterDynamics.SystemState — Method
SystemState(; bus_voltage=800.0, soc=0.5, controller_integrator=0.0)

Create a typed state object.

PowerConverterDynamics.battery_current_limits — Method
battery_current_limits(soc, params) -> (charge_limit, discharge_limit)

Return smooth SOC-dependent current limits in amperes.

PowerConverterDynamics.battery_ocv — Method
battery_ocv(soc, params) -> Real

Battery open-circuit voltage model, using a linearized OCV-SOC relation and smooth SOC limiting for numerical robustness.

PowerConverterDynamics.converter_efficiency — Method
converter_efficiency(source_voltage, source_current, bus_voltage, params) -> Real

Converter efficiency model with constant, linear, quadratic current losses plus extra loss sensitivity to source-vs-bus voltage mismatch.

PowerConverterDynamics.dynamics — Method
dynamics(state, params, input) -> (StateDerivative, FlowSnapshot)

Compute state derivatives and accompanying algebraic flow values.

PowerConverterDynamics.evaluate_flows — Method
evaluate_flows(state, params, input) -> FlowSnapshot

Compute algebraic currents, powers, efficiency, and net bus current imbalance for one operating point.

PowerConverterDynamics.simulate — Method
simulate(state0, params; kwargs...) -> SimulationResult

Run a time-domain simulation using fixed-size steps and user-provided profiles.

Each profile kwarg can be either:

  • A constant real value.
  • A callable f(t) returning the value at time t.

Keyword arguments:

  • tspan=(0.0, 1.0)
  • dt=1e-3
  • source_voltage=params.reference_bus_voltage
  • source_current_command=0.0
  • source_current_available=params.source_current_limit
  • load_current=0.0
  • battery_current_bias=0.0
PowerConverterDynamics.step — Method
step(state, params, input, dt) -> (next_state, flow)

Advance one explicit-Euler step with smooth state limiting.
